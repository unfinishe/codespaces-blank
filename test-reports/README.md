# Test reports

One Markdown file per release, auto-generated by `scripts/release-local.sh`.

## Naming convention

```
v<versionName>-<YYYYMMDD-HHmmss>.md
```

Example: `v1.0.5-20260415-143200.md`

## What is committed here

- A human-readable Markdown summary of all JUnit test results at release time.
- One file per released version — never edited manually after creation.

## What is NOT committed here

- Raw Gradle HTML or XML test reports (those stay in `build/`, which is gitignored).
- Any binary or generated file that is not a `.md`.

## Rationale

See `arc42/09_architecture_decisions.md` → ADR-010.
